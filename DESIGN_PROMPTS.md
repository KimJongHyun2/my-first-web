# Design Prompts — 실습용 프롬프트 모음 (Ch7.7)

이 파일은 **7.7 디자인 프롬프트 전략**을 실제로 적용할 수 있게 정리한 템플릿입니다.  
Copilot Chat에 복사/붙여넣기하면 바로 사용 가능합니다.

---

## 프롬프트 1: 초보자용 기본 프롬프트 (7.7.4 기반)

**상황**: 페이지의 레이아웃/색상/여백을 전반적으로 정리하고 싶을 때

```
#file:copilot-instructions.md #file:ARCHITECTURE.md

이 페이지를 개인 블로그답게 보기 좋게 정리해줘.

원하는 느낌:
- 깔끔하고 읽기 편한 블로그
- 배경은 밝게 유지
- 본문 글자는 선명하게
- 주요 버튼만 눈에 띄도록
- 지나친 그라디언트나 그림자 없음

조건:
- 나는 Tailwind나 shadcn/ui 클래스를 잘 모른다
- 너가 적절한 컴포넌트와 클래스를 골라줘
- 기존 기능(네비게이션, 데이터)은 그대로 유지해

수정 후 설명:
- 어떤 shadcn/ui 컴포넌트를 사용했는지
- 주요 Tailwind 클래스가 뭐하는 건지
- 수정한 파일 경로
```

## 프롬프트 2: 레퍼런스 제시 + 제약 조건 (전략 1+2)

**상황**: 특정 스타일이나 서비스를 참고하고 싶을 때

```
이 블로그 페이지를 "Medium 스타일" 로 만들어줘.

특징:
- 깔끔한 타이포그래피 (읽기 편한 폰트 크기)
- 여백이 충분함
- 최소한의 컬러 (흑백 + 1가지 강조색)
- 카드보다는 레이아웃 공간 활용

제약:
- shadcn/ui Card, Button, Input만 사용
- 디자인 토큰 준수 (#file:copilot-instructions.md 참고)
- 모바일(375px)에서 여전히 읽기 좋아야 함
```

## 프롬프트 3: 반복 다듬기 (전략 3)

**상황**: 한 번에 완벽한 결과보다는 단계별로 수정하고 싶을 때

**1차 요청** (기본 구조):
```
포스트 목록 페이지를 shadcn/ui Card로 정리해줘.

포함할 것:
- 포스트 제목
- 작성 날짜
- 본문 미리보기
- "읽기" 버튼

카드는 깔끔하게, 리스트는 1열 스택으로.
```

**2차 요청** (첫 결과 보고 수정):
```
좋아. 이제 다음을 수정해줘:
- 카드 간 여백을 더 넓게 (공간감 있게)
- 포스트 제목 폰트를 좀 더 크게
- 작성 날짜를 회색으로 연하게 표시

기존 기능은 유지.
```

**3차 요청** (추가 조정):
```
이제 반응형 추가해줘:
- 데스크톱(md 이상): 2열 그리드
- 모바일: 1열 유지

grid-cols-2, md:grid-cols-1 이런 식으로.
```

## 프롬프트 4: 부정 프롬프트 + 역할 부여 (전략 4+5)

**상황**: 과하게 꾸며진 디자인을 피하고 싶을 때

```
미니멀리스트 UI 디자이너 입장에서 이 페이지를 정리해줘.

금지사항:
- 그라디언트 사용 금지
- 그림자 과하게 금지 (shadow-sm 정도만)
- 애니메이션 불필요한 것 금지
- 색상 4가지 이상 섞지 마
- 테두리(border)는 필요한 곳만

사용할 것:
- 충분한 여백(padding, margin)
- 명확한 타이포그래피 계층
- 1~2가지 강조 색상만
- shadcn/ui 기본 컴포넌트

목표: "덜할수록 낫다" 느낌
```

## 프롬프트 5: 스크린샷 기반 수정 (검증 + 반복, 7.8.4 미리보기)

**상황**: 코드는 있는데 실제 화면이 어색할 때

**전제**: 먼저 `npm run dev`로 브라우저에서 화면을 보고, 데스크톱/태블릿/모바일 스크린샷을 찍은 후

```
[데스크톱 스크린샷, 태블릿 스크린샷, 모바일 스크린샷을 첨부하고]

이 스크린샷들을 보고 문제를 찾아줘.

확인할 것:
1. 텍스트가 겹치거나 잘리는 부분?
2. 버튼/카드 정렬이 어색한 부분?
3. 모바일에서 터치하기 어려운 요소?
4. 디자인 토큰(색상/간격)과 다른 부분?
5. 줄바꿈이 어색한 부분?

문제가 있으면:
- 어느 화면에서 뭐가 문제인지 지적
- 기존 기능 유지하면서 최소 수정
- 수정 이유 간단히 설명
```

## 프롬프트 6: 컴포넌트 적용 (이미 있는 페이지 개선)

**상황**: 기존 페이지에 shadcn/ui를 적용하고 싶을 때

```
#file:ARCHITECTURE.md #file:copilot-instructions.md

현재 페이지에 shadcn/ui 컴포넌트를 자연스럽게 적용해줘.

대체할 것:
- 글 목록 → Card로 표현
- 버튼 → Button 컴포넌트
- 입력창 → Input 컴포넌트
- 중요한 확인 팝업 → Dialog 컴포넌트

규칙:
- ARCHITECTURE.md의 컴포넌트 계층 참고
- copilot-instructions.md의 Design Tokens 사용
- 기존 데이터/기능은 100% 유지
- 스타일만 바뀌고 동작은 그대로

수정 후 알려줄 것:
- 변경한 파일 목록
- 사용한 shadcn/ui 컴포넌트
- 주의사항 (있으면)
```

## 프롬프트 7: 반응형 검증 (모바일 중심)

**상황**: 모바일에서 제대로 보이는지 확인하고 싶을 때

```
이 페이지가 모바일(375px)에서 제대로 보이도록 해줘.

체크할 것:
- 텍스트 크기: 본문 16px 이상
- 버튼: 터치 영역 44px × 44px 이상
- 여백: 위아래 좌우 충분한 padding
- 이미지: 컨테이너 너비를 벗어나지 않음
- 입력창: 터치하기 편한 높이(최소 44px)

Tailwind 조건:
- md: 이상에서만 그리드 2열
- sm 이하에서는 1열 유지
- md: 프리픽스로 데스크톱 규칙만 적용
```
